# 1. چجوری می تونید کمک کنید؟

حتی ریپورت یک باگ کوچیک هم کلی به من کمک می کنه.

چه کد، ریپورت باگ، ایده جدید، بهبود مستندات و داکیومنت یا یک بازخورد ساده. **هر چیزی که تو بهتر شدن این پروژه کمک بکنه.
**

**پس هر گونه کمک، از هر نوعی، فوق العاده کمک می کنه.**

# 2. قبل از شروع

برای جلوگیری ازهدر رفتن تایم خودتون (و من):

1. **همیشه اول [issue های باز](https://github.com/notMehrzad/Amelie-discord-bot/issues) رو چک کنید. ممکنه یک نفر همین
   الانش در حال کار کردن روی موضوعی باشه که توی ذهن شما هست.**

2. **پیشنهاد می شه که اول یک [issue باز کنید](https://github.com/notMehrzad/Amelie-discord-bot/issues/new) و صبر کنید تا
   مطمعن بشید کاری که می خواید انجام بدید لازم هستش. این کار از احتمال کار کردن روی موضوعی که ممکن هستش مناسب املی نباشه
   و رد بشه رو کم می کنه.**

   > استثنا: رفع اشکالات کوچیک مثل تایپو، غلط نوشتاری یا املایی، فرمت بندی کد یا تغییرات کوچیک داکیومنت **نیازی به باز
   > کردن issue ندارند.** می تونید مستقیما یک PR ثبت کنید.

## 2.1 فرایند اضافه کردن کامند ها / ویژگی های جدید

1. **یک [issue جدید](https://github.com/notMehrzad/Amelie-discord-bot/issues/new) با برچسب `enhancement` باز کنید.**

2. **به بهترین شکل ممکن توصیفش کنید.** بگید:
   - ایده اصلی این کامند/ویژگی چیه
   - این کامند/ویژگی قراره چیکار کنه
   - خروجی مد نظر چیه
   - چرا می تونه برای بقیه مفید باشه
   - مثال از نحوه استفادش

3. **منتظر بمونید. من بررسی می کنم که آیا ایده شما بدرد املی می خوره و می تونه برای بقیه مفید باشه یا نه. بعدش، شما (یا
   کس دیگه) می تونید روش کار کنید!**

## 2.2 وقتی یک Issue باز می کنید چه اتفاقی می افته؟

وقتی شما یک issue باز و ثبت می کنید، من سعی می کنم که در طی چند روز بررسیش کنم.

**اگه issue شما بعد از بررسی تایید بشه و مناسب املی و مفید باشه، issue باز می مونه و شما (یا کس دیگه) می تونید روش کار
کنید.**

**اگه issue شما به هر دلیلی تایید نشه، من issue رو می بندم.**

# 3. شروع به کار

**بهترین کار این هستش که برای حفظ کیفیت کد، راهنمایی زیر رو دنبال کنید.**

**پیشنیاز های زیر باید اول نصب شده باشن:**

- `Python 3.8.1` یا بالاتر
- `pip`
- `git`

## 3.1 راه اندازی ریپو

1. **این ریپو رو فورک کنید.**

2. **فورکتون رو کلون کنید.**

   ```bash
   git clone https://github.com/YOUR-USERNAME/Amelie-discord-bot.git
   cd Amelie-discord-bot
   ```

3. **ریپو اصلی رو به عنوان upstream اضافه کنید.**

   ```bash
   git remote add upstream https://github.com/notMehrzad/Amelie-discord-bot.git
   ```

4. **برنچ مربوط به ویژگی/کامند رو بسازید.**

   ```bash
   git checkout -b feature/your-feature-name
   ```

## 3.2 راه اندازی Virtual Environment

**پیشنهاد می شه که حتما یک محیط مجازی (Virtual Environment) بسازید تا کدها و پکیج هاتون در یک محیط ایزوله و جدا نگه داری
بشن و سایر پکیج های سیستمتون تحت تاثیر قرار نگیرن.**

```bash
python -m venv .venv
.venv\Scripts\activate
```

## 3.3 نصب وابستگی ها

1. **وابستگی ها رو از `requirements.txt` نصب کنید.**

   ```bash
   pip install -r requirements.txt
   ```

2. **یک فایل `config.json` بسازید.**

   ```json
   {
     "TOKEN": "BOT_TOKEN",
     "ADMINS": ["ADMINS_ID"]
   }
   ```

   - `BOT_TOKEN` رو با توکن بات تستتون جایگذاری کنید.
   - `ADMINS_ID` رو با ID اکانت دیسکورد خودتون جایگذاری کنید.

## 3.4 بروز بمونید

**سعی کنید گهگاهی از [upstream](https://github.com/notMehrzad/Amelie-discord-bot/tree/main) (برنچ اصلی ریپو اصلی) پول و
بروزرسانی ها رو دریافت کنید. همچنین، اگه متوجه شدید که `requirements.txt` تغییر کرده، به این معنی هستش که بعضی وابستگی
ها و پکیج ها ممکنه اضافه، حذف و یا تغییر کرده باشن. باید طبق تغییرات انجام شده پکیج های خودتون رو بروز کنید.**

# 4. شروع به کدنویسی

**ما در حال حاضر از [Black Formatter](https://github.com/psf/black) به عنوان فرمتر اصلیمون استفاده می کنیم، چرا که این
فرمتر به صورت خودکار بخش هایی از کد که نیاز به فرمت مجدد دارن یا ندارن رو شناسایی می کنه.**

**Type hint ها و مستندسازی در حین کدنویسی به شدت پیشنهاد می شه.**

**پس شروع به کدنویسی کنید و مطمعن بشید که کد شما مطابق استایل و فرمت گفته شده هستش.**

## 4.1 تغییرات رو تست کنید

**بات تستتون رو اجرا کنید و ببینید که آیا همه چیز درست کار می کنه یا نه.**

**تست شما باید شرایط زیر رو بگذرونه:**

- بات تست شما باید بدون ارور اجرا بشه

- تغییرات شما طبق چیزی که انتظار می ره کار می کنن

- کامند های غیر مرتبط همچنان کار می کنن

- بات تست شما کرش نمی کنه

## 4.2 قبل از کامیت اینارو چک کنید

- **کد شما مطابق فرمت و استایل ما هستش**

- **کد شما کامنت، پرینت های دیباگ و ... اضافی و غیر ضروری نداره**

- **همه چیز رو تست کردید**

- **فایل های ناخواسته رو کامیت نمی کنید. فایل `.gitignore` کامل و بروز هستش**

## 4.3 کامیت کنید

**بعد از تمام موارد گفته شده، تغییرات رو کامیت کنید و مطمعن بشید که یک پیغام کامیت تمیز و کامل می نویسید.**

فرمت پیغام کامیت:

- feat: → ویژگی یا کامند جدید

- fix: → باگ فیکس

- docs: → تغییر مستندات و داکیومنت

> در صورت لزوم می تونید توضیحات طولانی تری بنویسید.

# 5. به فورکتون پوش کنید و یک Pull Request باز کنید

**بعد از اینکه همه چیز رو تست کردید، و فقط اون موقع، می تونید تغییرات رو به فورکتون پوش کنید و یک PR به ریپو اصلی باز
کنید.**

## 5.1 بعد از اینکه PR ثبت می کنید چه اتفاقی می افته؟

1. **معمولا، PR شما توسط من در طی چند روز بررسی می شه.**

2. **ممکنه از شما خواسته بشه که در کدتون تغییراتی ایجاد کنید.**

3. **بعد از اینکه شما تمام تغییرات لازم رو طبق راهنمایی گفته شده انجام دادید، PR شما تایید و مرج میشه!**

## 5.2 اگه PR شما رد شد

**دلایل رایج برای اینکه PR شما رد شده می تونه شامل موارد زیر باشه:**

- **شما پروسه issue باز کردن رو انجام ندادید و یکی همین الانش یک PR مثل مال شما رو ثبت کرده**

- **کد شما تست ها رو رد نکرده**

- **کد شما مناسب املی نیست**

# 6. نیاز به کمک دارید؟

جایی گیر کردید؟ سوالی دارید؟

**یک issue با برچسب `question` باز کنید.**

> **قبل از اینکه چیزی بپرسید:** [issue های باز](https://github.com/notMehrzad/Amelie-discord-bot/issues) و همین داکیومنت
> رو مطالعه کنید. جواب سوالتون ممکنه همین الانش موجود باشه.
